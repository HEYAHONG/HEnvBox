# 说明

本目录主要用于辅助实现Z80模拟功能，Z80的主要实现代码见[../../h3rdparty/3rdparty/Z80/](../../h3rdparty/3rdparty/Z80/)，核心代码来源于[https://github.com/redcode/Z80.git](https://github.com/redcode/Z80.git)。

Z80是由美国Zilog公司推出的8位微处理器。

其特点如下：

- 兼容8080指令集
- 16位地址
- 两套寄存器
- 三种中断模式

# 编译器/开发环境 

由于Z80应用极广，编译器与开发环境也相对丰富。此章节仅描述本工程使用的编译器与开发环境。

- [SDCC](https://sdcc.sourceforge.net/)：支持Z80的编译器。
- [z88dk](https://z88dk.org)：基于Z80的机器的开发工具，包括C编译器、汇编器、库。其中C编译器支持多种模式，包括但不限于SDCC。.
- [z80pack](https://github.com/udo-munk/z80pack.git):Z80模拟器。包含一个汇编器。

# 模型

## 地址空间

Z80为8位处理器，采用16位地址空间，可寻址64KB空间。

Z80的内存空间与IO空间相互独立，即用于可使用64KB的内存空间与64KB的IO空间。

64KB的内存空间可映射为ROM、RAM、部分外设（如显示内存）。

64KB的IO空间可映射为外设寄存器：

| 地址范围   | 关键字(SDCC)     | 说明         |
| ---------- | ---------------- | ------------ |
| [0 0xFF]   | `__sfr`          | 8位地址空间  |
| [0 0xFFFF] | `__banked __sfr` | 16位地址空间 |

## 中断

Z80的中断分为不可屏蔽中断与可屏蔽中断。

## 不可屏蔽中断

直接跳转至`0x0066`地址。

使用SDCC编写不可屏蔽中断（NMI）服务函数需要指定`__critical __interrupt`关键字。

## 可屏蔽中断

| 模式 | 说明                                | 备注                                                         |
| ---- | ----------------------------------- | ------------------------------------------------------------ |
| 0    | 从数据总线读取指令（单字节）        | 通常为复位指令（`rst 0`～`rst 7`,分别对应地址 0x0000~0x0038）。 |
| 1    | 直接跳转至`0x0038`地址              | 同模式0的`rst 7`对应的地址                                   |
| 2    | 中断向量表（最多支持128个中断向量） | 向量表16位地址的高字节由I寄存器（复位时为0）提供，低字节由外设通过数据总线提供。 |

使用SDCC编写可屏蔽中断（INT）服务函数需要指定`__interrupt`关键字。

中断模式的切换需要使用汇编指令`im n`(n为对应的中断模式)

注意：

- SDCC不生成中断向量表。可使用汇编构造中断向量表或者在对应位置添加跳转指令。
- Z80的默认模式为模式0。

## 复位

Z80复位时PC默认为0x0000。
